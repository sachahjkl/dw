## Recommendation

Intégrer dans `dw` les idées de `S:\jb-agent-tools` qui renforcent la séparation déjà en place entre :
- actions déterministes CLI
- raisonnement/orchestration IA

Le bon axe n'est pas de copier `jb-agent-tools`, mais de :
- porter ses garde-fous et ses contrats utiles dans les commandes `dw`
- enrichir les contextes/templates agents de `dw`
- garder `dw` comme unique propriétaire des opérations Git/ADO/workspace

Point d’intégration recommandé :

## What Not To Import Directly

Ne pas reprendre tel quel depuis `jb-agent-tools` :
- la gestion git manuelle des branches/front/back
- les instructions de commit/push manuels
- le fichier de config séparé `ado.config.json`
- les conventions de branchement concurrentes à `dw`

Raison : `dw` possède déjà les rails déterministes pour :
- workspaces
- worktrees
- naming
- commit/push/PR
- ADO lifecycle

## Suggested Execution Order

1. Ajouter la sortie structurée sur `dw ado context/work-item`
2. Ajouter `task preflight` avec quelques checks ADO simples mais utiles
3. Générer les handoff files workspace
4. Faire consommer la synthèse structurée par `task finish`
5. Étendre `workflow.json` seulement après stabilisation des besoins réels
