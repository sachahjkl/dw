# Plan: corriger `dw task open --continue`

## Problème

`dw task open --continue` utilise aujourd'hui `--continue` pour deux sens différents :

1. sélectionner le dernier workspace `dw` créé ou mis à jour ;
2. transmettre `Continue=true` à l'adaptateur agent.

Pour OpenCode, `Continue=true` produit `opencode -c <workspace>`, ce qui tente de reprendre une session OpenCode existante. Après `dw task start`, surtout quand les worktrees/branches existent déjà, il peut ne pas y avoir de session OpenCode à reprendre. OpenCode échoue alors alors que l'intention utilisateur est simplement d'ouvrir le dernier workspace.

## Approche recommandée

Séparer la notion de sélection du workspace et la notion de reprise de session agent.

### 1. Modifier le modèle d'ouverture agent

Fichiers :

- `src/Dw.Cli/Agents/AgentAdapter.cs`
- `src/Dw.Cli/Workspaces/WorkspaceOpenService.cs`
- `src/Dw.Cli/Cli/SystemCommandLineApp.cs`

Ajouter un booléen distinct, par exemple `ResumeSession`, dans `WorkspaceOpenOptions` et `AgentOpenRequest`.

- `Continue` garde son sens CLI actuel : choisir le dernier workspace.
- `ResumeSession` indique explicitement qu'il faut demander à l'agent de reprendre une session.

### 2. Ajouter une option CLI explicite

Ajouter une option de `task open`, par exemple :

```powershell
dw task open --continue
```

ouvre le dernier workspace sans reprise de session.

```powershell
dw task open --continue --resume
```

ouvre le dernier workspace en demandant à l'agent de reprendre une session existante.

Nom d'option recommandé : `--resume`.

Raison : plus clair que `--continue-session`, cohérent avec l'intention agent, et évite de casser le sens déjà installé de `--continue`.

### 3. Adapter OpenCode

Fichier :

- `src/Dw.Cli/Agents/OpenCodeAgentAdapter.cs`

Changer :

```csharp
request.Continue ? ["-c", request.Workspace] : [request.Workspace]
```

vers :

```csharp
request.ResumeSession ? ["-c", request.Workspace] : [request.Workspace]
```

Ainsi :

- `dw task open --continue` lance `opencode <workspace>`.
- `dw task open --continue --resume` lance `opencode -c <workspace>`.

### 4. Adapter les autres agents

Fichiers probables :

- `src/Dw.Cli/Agents/ClaudeAgentAdapter.cs`
- `src/Dw.Cli/Agents/CursorAgentAdapter.cs`
- `src/Dw.Cli/Agents/CodexCliAgentAdapter.cs`
- `src/Dw.Cli/Agents/CopilotAgentAdapter.cs`

Remplacer leur utilisation de `request.Continue` par `request.ResumeSession` si cette valeur est transmise comme option de reprise de session agent.

### 5. Mettre à jour le message de `task start`

Fichier :

- `src/Dw.Cli/Workspaces/TaskStartService.cs`

Conserver :

```text
Prochaine etape:
  dw task open --continue
```

Ne pas mentionner `--resume` dans le flux par défaut.

Ajouter éventuellement une note courte ailleurs dans l'aide : `--resume` sert uniquement à reprendre une session agent existante.

### 6. Tests à modifier/ajouter

Fichier principal :

- `tests/Dw.Cli.Tests/WorkspaceOpenServiceTests.cs`

Modifier le test existant :

- `Open_sets_opencode_config_and_continue_flag`

Il doit devenir deux tests :

1. `Open_continue_selects_latest_workspace_without_resuming_agent_session`
   - appeler `WorkspaceOpenService.Open(... Continue: true, ResumeSession: false)`
   - attendre `opencode` avec arguments `[workspace]`, pas `['-c', workspace]`

2. `Open_resume_passes_agent_resume_flag`
   - appeler avec `ResumeSession: true`
   - attendre `opencode` avec arguments `['-c', workspace]`

Ajouter/adapter tests CLI dans :

- `tests/Dw.Cli.Tests/AppTests.cs`

Vérifier que `task open --help` expose `--resume`.

### 7. Version et commit

Bumper `VERSION` dans le même commit, comme demandé.

Message de commit recommandé :

```text
fix: separer ouverture workspace et reprise agent
```
